# Prompt Templates

## 1. 项目理解 Prompt

```text
请先阅读我上传的客户 PRD / 需求文档 / 会议纪要 / 项目资料，先不要生成 PPT，也不要生成图片。

你的任务是先理解这个项目，请输出：
1. 项目背景；
2. 客户当前需求；
3. 目标用户 / 使用角色；
4. 核心功能；
5. 项目整体价值；
6. 适合在方案汇报 PPT 中重点呈现的内容；
7. 哪些内容是技术细节，哪些内容适合转化成客户能听懂的业务价值。

注意：现在只做项目理解，不做 PPT，不生成图片。

最后请给我选择：
A. 项目理解正确，生成文本版 PPT 大纲
B. 项目理解有偏差，我要补充 / 修改
C. 重新梳理项目理解
D. 由 ShirliY AI 做决定
```

## 2. 文本版 PPT 大纲 Prompt

```text
请基于客户资料和刚才确认的项目理解，设计一份 12～15 页的方案汇报 PPT 文本版大纲。

注意：
1. 不要生成 PPT；
2. 不要生成图片；
3. 只输出大纲；
4. 这份大纲后续会用于直接生成一张总览预览图和逐页 16:9 PPT 单页预览图；
5. 每页需要包含：页面标题、页面目的、核心内容、建议版式、客户能理解的价值。

这份 PPT 的定位是客户业务洽谈 / 方案汇报 / 合作沟通使用，不是研发内部评审文档。

输出格式：
第 X 页：页面标题
页面目的：
核心内容：
建议版式：
客户能理解的价值：

最后请给我选择：
A. 大纲确认，进入风格确认
B. 调整页数 / 顺序 / 重点内容
C. 重新生成一个大纲方向
D. 由 ShirliY AI 做决定
```

## 3. 风格确认 Prompt

```text
现在内容大纲已确认，请选择本次 PPT 的视觉风格方向：

A. 采用公司风格
使用 GeekDance Official Business Style。
默认使用固定模板背景「assets/geekdance-default-background.jpg」作为 16:9 页面底板。
页面是在模板上排版，不是自由生成整页；所有新增文字、图表、卡片和视觉元素都叠加在该模板之上。
右上角 Logo 不可编辑，不允许移动、缩放、改色、重绘、替换或加特效。
顶部大字水印保持模板原样，不重新设计。
视觉参考 GeekDance 公司介绍风格：白色商务科技风、红色点缀、圆角卡片、轻阴影、图表化表达。

B. 公司风格 + 客户风格结合
请上传客户 Logo、品牌色、官网截图、历史 PPT 或风格要求。
我会在 GeekDance 公司风格基础上，融合客户品牌元素。

C. 完全客户化风格
请上传客户 Logo、VI、官网截图、历史 PPT 或明确风格要求。
我会完全按照客户风格执行，不默认使用 GeekDance 视觉体系。

D. 由 ShirliY AI 做决定
我会根据客户行业、项目性质、汇报对象和资料完整度，自动判断采用公司风格、融合风格，还是完全客户化风格。
```

## 5. 总览预览图生成指令

```text
现在请先不要逐页生成。

请根据刚才确认的 PPT 大纲和视觉风格，直接生成一张「总览预览图」。

要求：
1. 这张图不是多页拼贴，也不是多页并排合集；
2. 不要使用多页网格并排展示；
3. 这张图是一张完整的大图预览，用来体现整套 PPT 的整体风格、版式系统、气质方向；
4. 这张图只用于确认整体视觉方向，不代替逐页 16:9 单页预览图；
5. 如果采用公司风格，必须使用 GeekDance Official Business Style；
6. 如果采用公司风格，必须以 assets/geekdance-default-background.jpg 为固定模板背景，页面是在模板上排版，不是自由生成；
7. 如果采用公司风格，右上角 Logo 不允许任何变动；
8. 不要生成 PPT 文件，不要生成 PPT 源文件，不要输出 PPT 组装指令；
9. 只生成一张总览预览图片；
10. 默认不要输出生图 Prompt。只有当用户明确要求时，才额外输出该总览图对应的 Prompt。

最后请给我选择：
A. 生成第一张
B. 一次性生成所有
C. 重新修改风格

选项逻辑：
1. A 代表基于当前已确认的风格和大纲，开始生成第 1 张 16:9 PPT 单页预览图；
2. B 代表基于当前已确认的风格和大纲，直接一次性生成整套所有页面的 16:9 单页预览图；
3. C 代表返回风格调整流程，重新确认风格方向，然后重新生成新的总览预览图。
```

## 6. 逐页 16:9 PPT 单页预览图生成指令

```text
现在按照刚才确认的整体 PPT 总览预览图，开始逐页直接生成 16:9 PPT 单页预览图。

要求：
1. 每次只生成一张 16:9 横版 PPT 单页预览图；
2. 按文本版 PPT 大纲中的顺序生成；
3. 保持总览预览图确定的整体风格、配色、版式系统和内容逻辑；
4. 每一页都必须是完整的 16:9 单页预览图图片；
5. 图片可以直接复制进 PPT 使用；
6. 不要生成 PPT 文件；
7. 不要生成多页拼图。
8. 默认不要输出该页 Prompt。只有当用户明确要求时，才额外输出该页对应的 Prompt。
9. 如果采用公司风格，优先把 assets/geekdance-default-background.jpg 作为页面底板，所有新增内容叠加在其上。
10. 不要重新生成背景，不要破坏模板现有元素。
11. 右上角 Logo 不可编辑：不移动、不缩放、不改色、不重绘、不替换、不加特效。
12. 不要生成新的英文 GeekDance logo，不要把原 Logo 替换成新 Logo。
13. 背景顶部的大字水印保持模板原样，不重新设计。

请直接生成第 X 页 16:9 PPT 单页预览图。

输出结构：
当前生成：第 X 页
页面标题：
页面目标：
核心内容：
视觉版式：
已生成：第 X 页 16:9 PPT 单页预览图

最后请给我选择：
A. 这一页确认，继续下一页
B. 修改这一页
C. 一次性全部生成

选项逻辑：
1. A 代表当前页通过，继续生成下一页；
2. B 进入当前页修改流程，询问用户要修改的内容；
3. C 代表用户确认当前风格和当前页方向，直接按当前已确认的风格、版式系统和内容逻辑，一次性生成剩余全部页面的预览图，然后进入整套复核或逐页微调；
4. 单页阶段不要展示“重新生成这一页预览图”或“由 ShirliY AI 做决定”。
```

## 7. 单页修改生成指令

```text
这一页整体风格保持不变，只修改以下内容：
1. XXX；
2. XXX；
3. XXX；
4. 其他内容和版式保持不变。

请直接重新生成这一页的 16:9 PPT 单页预览图。默认不要输出 Prompt；只有当用户明确要求时，才额外输出该页对应的 Prompt。

最后请给我选择：
A. 这一页确认，继续下一页
B. 修改这一页
C. 一次性全部生成

选项逻辑与单页生成后完全一致：A 继续下一页，B 继续修改当前页，C 一次性生成剩余全部页面。
```

```text

2. 不生成 PPT 文件；
3. 不输出 PPT 源文件；
4. 不输出 PPT 组装指令作为默认结果；
5. 只有用户明确要求时，才额外输出 Prompt 或 PPT 组装说明；
6. 输出给客户前进行最终人工审核。

同时输出最终复核清单。
```
