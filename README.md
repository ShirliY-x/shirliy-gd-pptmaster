# Shirliy｜极客PPT大师

一个给 **SE / 解决方案同事 / 商务同事** 使用的 Codex Skill，用来快速把客户资料整理成一套「客户能看懂、商务能拿去讲、视觉又像 GeekDance」的方案汇报 PPT 预览图。

它不是传统的“自动生成可编辑 PPT 文件”的工具，而是一个更稳的 **PPT 方案图生成工作流**：

客户资料理解
→ 文本版 PPT 大纲
→ 风格确认
→ 整套缩略总览预览图
→ 逐页 16:9 单页预览图
→ 人工复核与交付

简单说：

> 先帮你想清楚怎么讲，再帮你一页一页生成能看的 PPT 预览图。

---

## 这个 Skill 适合谁用？

### 适合 SE / 解决方案同事

当你拿到客户 PRD、会议纪要、需求文档、功能清单、报价要求之后，可以用它来快速梳理：

- 客户到底要解决什么问题；
- 这套方案应该怎么讲才有逻辑；
- 哪些技术内容要转成客户听得懂的业务价值；
- 一套方案 PPT 应该怎么分章节、怎么排页面。

### 适合商务 / BD 同事

当你需要给客户做方案汇报、初步提案、合作沟通、项目推进材料时，可以用它来快速生成：

- 客户方案汇报大纲；
- 商务化表达；
- 整套 PPT 缩略预览图；
- 逐页 16:9 PPT 页面预览图；
- 最终检查清单。

### 不适合直接拿来做什么？

它默认不做这些事情：

- 不默认生成可编辑 PPT 文件；
- 不默认直接跳过大纲就生图；
- 不默认一次性无脑生成整套页面；
- 不默认输出一堆 Prompt 让你自己复制。

它的重点是：**一步一步确认，让方案更稳。**

## 最推荐的使用流程

### Step 1｜先上传客户资料

可以上传或粘贴：

- 客户 PRD；
- 需求文档；
- 会议纪要；
- 功能清单；
- 报价要求；
- 客户官网截图；
- 原型图 / UI 图；
- 你自己整理的项目背景。

资料不完整也没关系，可以直接说：

资料还不完整，你先基于现有内容帮我判断还缺什么。

---

### Step 2｜先让它理解项目，不要急着做 PPT

正确指令示例：

请先理解这个客户项目，不要生成 PPT，也不要生成图片。
请输出：项目背景、客户需求、目标用户、核心功能、项目价值，以及适合在方案汇报里重点讲的内容。

它会先帮你整理项目理解。你确认后，再进入大纲。

---

### Step 3｜生成文本版 PPT 大纲

它会输出类似这样的结构：

第 X 页：页面标题  
页面目的：  
核心内容：  
建议版式：  
客户能理解的价值：

这一步很重要。因为客户方案 PPT 不是堆功能，而是要把事情讲清楚：

- 为什么要做；
- 做什么；
- 怎么做；
- 对客户有什么价值；
- 为什么 GeekDance 能做。

---

### Step 4｜确认风格

通常有三种选择：

A. 采用公司风格  
B. 公司风格 + 客户风格结合  
C. 完全客户化风格  
D. 由 ShirliY AI 做决定

如果是 GeekDance 主导的客户方案，建议优先选：

A. 采用公司风格

如果客户有明显品牌要求，比如 Logo、品牌色、官网风格、历史 PPT，可以选：

B. 公司风格 + 客户风格结合

---

## 公司风格怎么理解？

默认公司风格是 **GeekDance Official Business Style**。

核心视觉规则：

- 16:9 横版；
- 使用固定 GeekDance 背景母版；
- 右上角 Logo 保持不变；
- 顶部浅色 GeekDance 大水印保持不变；
- 白色、浅灰、淡粉白为主；
- 红色作为重点强调；
- 使用圆角卡片、轻阴影、红色线性图标、流程图、架构图、矩阵图；
- 整体要专业、干净、商务、科技感；
- 不要做成蓝色运维大屏风；
- 不要做成普通模板 PPT。

特别注意：

> 背景图不是“参考图”，而是固定母版。右上角 Logo 是锁定区域，不能重绘、不能改色、不能换位置、不能缩放、不能遮挡。

## 预览图怎么生成？

这个 Skill 现在默认是 **直接生成预览图**，不是只输出 Prompt。

### 总览阶段

它会生成一张整套 PPT 的缩略总览预览图，用来先看：

- 页数是否合理；
- 章节逻辑是否顺；
- 整体风格是否对；
- 每页版式有没有明显问题。

总览图里的每一页都必须保持 16:9，不能被压扁、拉伸或变形。

### 逐页阶段

确认总览方向后，它会逐页生成 16:9 单页预览图。

每生成一页后，只需要选择：

A. 这一页确认，继续下一页  
B. 修改这一页  
C. 一次性全部生成

这三个选项的意思是：

- **A**：这一页 OK，继续生成下一页；
- **B**：这一页要改，先改完再继续；
- **C**：当前风格和方向 OK，直接把剩余页面一次性生成出来。

## 给 SE 的实用建议

### 1. 不要一开始就说“帮我生成 PPT”

更好的说法是：

先帮我理解这个客户项目，然后输出适合商务汇报的 PPT 大纲。

### 2. 技术内容要让它翻译成业务价值

比如不要只写：

使用 RAG、Agent、微服务、向量库。

要让它转成：

帮助客户沉淀知识、自动处理任务、提升跨部门协作效率，并降低后续运营成本。

### 3. 客户方案要讲“价值”，不是只讲“功能”

每一页尽量回答一个问题：

- 客户为什么需要它？
- 这页想说服客户什么？
- 客户看完以后会不会更理解项目价值？

### 4. 关键信息必须人工复核

尤其是：

- 客户名称；
- 项目名称；
- 金额；
- 工期；
- 日期；
- 交付范围；
- 承诺内容；
- 是否误用了旧参考资料。

## 交付前检查清单

在给客户之前，至少检查一遍：

- [ ] 客户名称是否正确
- [ ] 项目名称是否正确
- [ ] 日期、金额、工期是否正确
- [ ] 页面比例是否为 16:9
- [ ] 公司背景母版是否正确
- [ ] 右上角 Logo 是否保持原样
- [ ] 是否有错别字
- [ ] 是否有逻辑不通的内容
- [ ] 是否误用了参考图里的旧内容
- [ ] 是否有过度承诺
- [ ] 是否适合客户业务洽谈场景
